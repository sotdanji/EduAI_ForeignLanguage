# EduAI 개발 계획서 (학생용 AI 영어 튜터)
> **"교육의 주도권을 학생에게"**

## [Goal Description]
현재 `BookReader` 프로젝트의 코어 엔진(OCR, 번역, TTS)을 기반으로, 학생들의 능동적 어학 학습에 특화된 별도의 독립 프로젝트인 **EduAI(가칭)**를 개발합니다. 출판사가 제공하는 '교사 중심의 일방향 스마트 교재'와 차별화하여, 학생이 언제 어디서든 모르는 영어 지문(교과서, 문제집, 핸드아웃 등)을 카메라로 찍어 스스로 분석하고 학습할 수 있는 **'개인화된 AI 튜터'**를 구축하는 것이 목표입니다.

## Proposed Features & Architecture

### 1. 핵심 기능 (Key Features)

*   **스마트 스캔 및 스크린샷 연동 (Smart Scan & Screenshot)**
    *   **카메라 직접 촬영**뿐만 아니라, 기기 내 **이미지 업로드 및 스크린샷 캡처** 기능을 지원하여 태블릿 화면의 디지털 교과서나 인강 화면도 즉시 불러와 분석 가능.
    *   **선택 영역 자르기(Crop) 지원:** 전체 화면을 캡처하더라도 앱 내에서 모르는 문장이나 문제 영역만 드래그하여 자를 수 있는 기능을 제공하여 AI의 분석 정확도 향상.
    *   학생이 교재나 필기를 캡처/촬영하면 Gemini API(VLM/OCR)가 텍스트와 손글씨를 추출하며, 단순 전사가 아닌 **오탈자 보정, 문맥 파악, 필기 오답 분석**을 동시에 수행.
*   **AI 퀴즈 자동 생성 (Quiz Generator)**
    *   스캔한 지문을 바탕으로 즉석에서 내용 이해를 확인하는 퀴즈(True/False, 빈칸 채우기, 객관식 내용 일치)를 생성하여 인터랙티브 UI 제공.
*   **단어장 & 문법 노출 (Auto Vocabulary & Grammar)**
    *   지문 내 핵심 영단어와 타겟 문법(예: 현재완료, 관계대명사)을 AI가 추출하여 하이라이트.
    *   클릭 시 발음 듣기 및 '나만의 단어장' 저장 기능.
*   **롤플레잉 및 발음 평가 (Interactive Speaking)**
    *   대화문 스캔 시 역할을 나누어 실시간 원어민 튜터(Gemini Multimodal Live API)와 대화 연습.
    *   기존의 느린 STT/TTS 단계를 제거하고, 음성-음성 직결(Voice-to-Voice)을 통해 억양 교정 및 즉각적인 피드백 제공.

### 2. 기술 스택 (Tech Stack)

*   **Front-end & Back-end**: Streamlit (태블릿/PC 가로형 분할 레이아웃 최우선 적용, 스마트폰 반응형 지원 및 PWA)
*   **Language**: Python 3.x
*   **AI/LLM**: Google Gemini 1.5 Flash (OCR, 번역, 퀴즈 출제, 문법 분석)
*   **Audio & Interactive**: `Gemini Multimodal Live API` (실시간 음성-음성 통번역 및 튜터링 처리)
*   **Database**: SQLite (단어장, 퀴즈 성적 등 로컬 프로파일링)

## Development Roadmap (개발 단계)

### Phase 1: Core Engine Refactoring (준비 단계)
*   기존 `BookReader`에서 OCR, 번역, TTS 모듈을 재사용 가능하도록 함수화 및 패키지화.
*   `EduAI` 전용 프로젝트 폴더 및 가상 환경 세팅.

### Phase 2: EduAI MVP 개발 (태블릿 분할 뷰 중심)
*   태블릿의 넓은 화면에 최적화된 가로형 2단 분할 뷰(좌측 이미지 뷰어 + 우측 AI 튜터) UI 구현.
*   **퀴즈 생성 프롬프트** 엔지니어링 및 터치 친화적 UI 연동 (T/F, 객관식).
*   핵심 영단어 자동 추출 및 로컬 단어장(SQLite) 연동.

### Phase 3: Advanced Interactive Features (고도화 단계)
*   STT 연동을 통한 발음 녹음 및 정확도 평가 모듈 개발.
*   대화문 인식 및 롤플레잉(Role-play) 모드 추가.
*   학생의 누적 학습 데이터(오답 노트, 단어 암기량) 대시보드 시각화.

## Verification Plan

### Automated Tests
*   `pytest`를 활용하여 Gemini API가 주어진 텍스트에서 정상적으로 JSON 형태의 퀴즈 데이터(문제, 보기, 정답)를 생성하는지 파싱 테스트.
*   단어장 DB Create/Read/Update/Delete (CRUD) 유닛 테스트.

### Manual Verification
*   실제 중학교 영어 교과서 페이지를 카메라로 촬영(또는 PDF 캡처 이미지 사용)하여:
    1. 오탈자 없이 텍스트가 보정되어 추출되는지 확인.
    2. 생성된 퀴즈의 난이도와 정답이 원문과 일치하는지 확인.
    3. 사용자 음성을 녹음하여 STT가 올바르게 텍스트로 변환하고 채점하는지 확인.
